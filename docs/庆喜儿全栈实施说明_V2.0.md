# 庆喜儿全栈实施说明 V2.0

## 1. 技术栈纠偏

V1.0/V1.1 方案指定 API/BFF 使用 Node.js 20、NestJS 和 TypeScript。V2.0 已删除 Python 作为最终主实现的口径，形成统一 npm workspace：服务器、uni-app 小程序、Vue 管理后台和共享契约。

## 2. 服务器实现

服务器采用模块化单体，围绕用户、设备、语音、提醒、关系消息、内容、声音、任务、订阅、订单、OTA 和运营能力组织。对外提供 107 个 REST 路由与 `/ws/v1/voice`。

语音链路：本地 “阿西” 唤醒 → WSS 音频 → Fun-ASR 适配 → 方言路由 → 幼儿安全库/成人 LLM → CosyVoice 克隆音色 → MQTT 动作与设备播放。高置信度锁定、中置信度会话滞回、低置信度使用设备兜底；泛吴语显式映射上海话并返回降级原因。

数据层提供内存测试实现与 MongoDB 适配；事件层提供内存实现与 MQTT 适配。生产配置通过环境变量注入，不将密钥写入仓库。

## 3. 小程序实现

uni-app 小程序使用 Vue 3 + Pinia，提供 17 个页面。BLE 服务实现扫描、连接、凭据校验、Wi-Fi 写入和失败状态。API 客户端统一注入 Bearer Token、Trace ID，解析标准错误体。用户无须手动切换方言，仅查看识别状态和降级提示。

## 4. 管理后台实现

后台包含 13 个运营页面。管理员写操作携带 MFA 标识，服务端再次校验；内容、OTA、唤醒词、用户状态等敏感操作写入审计日志。页面按路由懒加载，API 统一封装。

## 5. 测试策略

- 服务器：Jest + Supertest + `ws`，包含应用、基础设施、错误、安全、供应商请求和完整 WSS 流；
- 小程序：Vitest，覆盖 HTTP、BLE、Pinia、视图模型和全量 API 包装器；
- 后台：Vitest，覆盖 HTTP/MFA、全量管理 API、状态与路由；
- 构建：TypeScript、NestJS、uni-app `mp-weixin`、Vite；
- 进程：实际启动 `server/dist/main.js`，执行 HTTP/WSS 冒烟；
- CI：三端独立 Job，避免单个工具链故障相互污染。

## 6. 发布门禁

代码门禁：三端类型检查、单测、覆盖率和构建全部通过，生产依赖审计无高危/严重问题。环境门禁：正式语音语料、微信真机/BLE、支付、MongoDB/EMQX、并发、渗透、ESP32/OTA 和 72 小时长稳完成后方可生产放行。
