1. 部署前提和责任边界

以下以“贵方/甲方”为需准备域名、服务器和第三方账号的一方。乙方/研发方默认负责应用部署包、数据库初始化脚本、接口配置说明、固件 OTA 包和联调支持。

| 关键前提：微信小程序合法域名、HTTPS 证书、COS/CDN 域名、MQTT/WSS 域名必须在联调前准备完成，否则设备配网、语音链路、上传下载和小程序真机调试会受阻。 |

2. 域名、DNS 与证书清单

| 项目 | 建议值 | 准备方 | 说明 |
| 主域名 | qingxier.com 或贵方自有域名 | 甲方 | 需完成实名/备案/解析权限；以下子域名均以示例主域名替换。 |
| API 域名 | api.qingxier.com | 甲方/运维 | 小程序 request 合法域名；HTTPS 443；对接后端 REST API。 |
| 语音 WebSocket 域名 | voice.qingxier.com | 甲方/运维 | 小程序/设备 socket 合法域名；WSS 443；承载实时语音。 |
| MQTT 域名 | mqtt.qingxier.com | 甲方/运维 | 设备 MQTTS 8883；可额外启用 WSS MQTT 8084/443。 |
| 运营后台域名 | admin.qingxier.com | 甲方/运维 | 运营后台和内容审核；建议加 IP 白名单/MFA。 |
| 资源/CDN 域名 | cdn.qingxier.com | 甲方/云账号 | 小程序 downloadFile 合法域名；OTA、表情、TTS、自定义内容下载。 |
| 上传域名 | upload.qingxier.com 或 COS 域名 | 甲方/云账号 | 小程序 uploadFile 合法域名；语音样本、自定义故事上传。 |
| 支付回调域名 | api.qingxier.com/payments/wechat/notify | 甲方/微信支付 | 微信支付商户平台配置回调 URL；需公网 HTTPS。 |
| 证书 | 通配符 *.qingxier.com 或逐域名单域证书 | 甲方/运维 | Nginx/负载均衡、EMQX、WSS 均需部署；注意续期告警。 |

2.1 小程序平台需配置的合法域名

| 小程序配置项 | 建议域名 |
| request 合法域名 | https://api.qingxier.com |
| socket 合法域名 | wss://voice.qingxier.com；如小程序直连 MQTT over WSS，则增加 wss://mqtt.qingxier.com |
| uploadFile 合法域名 | https://upload.qingxier.com 或 COS 自定义上传域名 |
| downloadFile 合法域名 | https://cdn.qingxier.com |

3. 服务器与云资源清单

图 1 建议生产部署拓扑：负载均衡、API、语音、EMQX、MongoDB、Redis、COS/CDN、监控

3.1 环境规划

| 环境 | 用途 | 最低资源 | 说明 |
| 开发环境 dev | 研发自测、接口联调、Mock 第三方 | 1 台 2C4G 或本地 Docker Compose | 可与测试资源复用，但数据必须隔离。 |
| 测试环境 test | 甲乙联调、真机调试、验收预演 | API 2C4G；EMQX 2C4G；MongoDB/Redis 2C4G；COS 测试桶 | 建议独立域名 test-api/test-mqtt/test-cdn。 |
| 生产环境 prod | 正式用户和设备接入 | 见 3.2 推荐规格 | 需高可用、备份、监控、告警、权限隔离。 |

3.2 生产资源推荐

| 资源 | 建议规格 | 用途 | 备注 |
| 负载均衡/WAF | 1 套 | 公网 HTTPS 443，转发 API/voice/admin | 建议云负载均衡 + WAF，证书统一托管。 |
| API/BFF 服务 | 2 台 4C8G 起 | Node.js/NestJS 容器；可水平扩展 | 初期可单台，正式上线建议双节点。 |
| AI/语音网关 | 1-2 台 4C8G 起 | WSS 音频、ASR/TTS/LLM 调度 | 如自建 ASR/TTS/LLM 需另配 GPU；MVP 推荐第三方 API。 |
| EMQX MQTT | 2 台 2C4G 起 | MQTTS 8883/WSS 8084；设备心跳与命令 | 设备量增加后扩为 3 节点集群。 |
| MongoDB | 云数据库或 3 节点副本集，4C8G 起 | 业务主库、对话、消息、记忆、内容元数据 | 生产建议托管版；启用备份和慢查询。 |
| Redis | 云 Redis 主从，2C4G 起 | 在线状态、会话、限流、队列、分布式锁 | 启用持久化和内网访问。 |
| COS 对象存储 | 按量 | OTA 固件、表情包、TTS 音频、自定义故事、声音样本 | 分 test/prod bucket；设置生命周期和防盗链。 |
| CDN | 按量 | 资源加速下载 | 绑定 cdn 子域名；OTA 可按区域加速。 |
| 日志与监控 | 云日志/Prometheus/Grafana/Sentry | 应用日志、MQTT 事件、AI 延迟、错误告警 | 需保留至少 30 天；验收长稳测试时查看。 |
| 备份存储 | 按量 | MongoDB 备份、配置备份、OTA 包归档 | 生产至少每日备份，保留 7-30 天。 |

3.3 网络与端口

| 端口 | 开放范围 | 用途 | 安全要求 |
| 443/TCP | 公网 | HTTPS REST、WSS 语音、小程序访问 | 开放给公网；建议走 LB/WAF。 |
| 8883/TCP | 公网或专线 | MQTTS 设备接入 | 设备证书/Token 鉴权；限制 MQTT ACL。 |
| 8084/TCP | 按需公网 | MQTT over WSS | 如果小程序或调试工具需要直连 MQTT。 |
| 22/TCP | 运维 IP | SSH 管理 | 仅堡垒机或固定 IP；禁止全网开放。 |
| 27017/TCP | 内网 | MongoDB | 仅 API/AI 服务访问。 |
| 6379/TCP | 内网 | Redis | 仅 API/AI/Worker 访问。 |
| 18083/TCP | 运维 IP | EMQX Dashboard | 启用强密码/MFA/IP 白名单。 |

6. 部署步骤

1. 完成主域名备案、DNS 解析、SSL 证书申请，并配置 API/voice/mqtt/admin/cdn/upload 子域名。

2. 创建 VPC、子网、安全组、负载均衡/WAF，按端口清单开放访问。

3. 创建 MongoDB、Redis、COS Bucket、CDN 加速域名和日志/监控资源。

4. 部署 EMQX，配置 TLS、认证、ACL、遗嘱消息、系统事件 Webhook 和监控。

5. 部署 API/BFF、AI/语音网关、Worker/调度服务，导入环境变量并执行数据库初始化脚本。

6. 配置微信小程序合法域名、AppID/AppSecret、支付回调和订阅消息模板。

7. 上传首版固件、表情包、幼儿内容库到 COS，创建 OTA 包记录。

8. 设备烧录生产固件，写入 SN/deviceSecret，完成 BLE 配网、MQTT/WSS 接入联调。

9. 执行接口自动化测试、MQTT Topic 测试、语音链路测试、OTA 回滚测试和 72 小时长稳测试。

10. 验收通过后切换生产域名和正式小程序版本，开启监控告警和备份策略。

7. 验收准备清单

| 验收项 | 准备内容 | 通过口径 |
| 蓝牙配网 | 小程序真机、测试 WiFi、设备 3 台以上 | 60 秒内成功，设备语音提示联网成功。 |
| 设备状态 | 电量、在线、模式、人格、固件版本 | 小程序修改后设备确认，云端刷新。 |
| 语音 AI | 安静/背景音场景、成人/幼儿模式、五种人格 | 响应延迟、唤醒成功率、误唤醒率达标。 |
| 提醒 | 喝水/吃药/周期提醒、二次确认、未确认递进 | 设备播报、微信通知、完成和解记录完整。 |
| 跨设备留言 | 两台绑定设备、文字/语音留言、离线/上线补发 | 在线设备 3 秒内提醒，已读回执正确。 |
| 自定义内容 | 问答 50 组、提醒 20 条、故事 ≤5 分钟、超限用例 | 容量限制、优先级、删除后失效达标。 |
| 声音克隆 | 20-30 秒样本、训练回调、人格绑定、删除 | 音色应用和数量限制符合套餐。 |
| OTA | 固件、表情包、幼儿内容库、损坏包回滚 | 版本更新、资源可用、失败回滚。 |
| 性能稳定 | 72 小时运行、1000 条自定义问答压力 | 无死机/泄漏；响应 ≤2 秒。 |
| 安全合规 | 未成年人数据、声音授权、支付、后台审计 | 授权文案、删除入口、日志和权限检查完成。 |

8. 上线风险与确认项

| 确认项 | 说明 |
| BOM 与固件版本 | 确认电池容量、GPIO 复用、硬件版本号、固件版本号和 OTA 兼容矩阵。 |
| 第三方 SLA | 确认 LLM/ASR/TTS/声音克隆的并发、超时、失败重试、账单告警和数据合规。 |
| 小程序审核 | 功能涉及 AI、儿童内容、付费订阅、声音克隆时，需预留审核周期和补充材料。 |
| 数据合规 | 幼儿版数据保存、家长查看对话、成人长期记忆、声音样本授权需法务确认。 |
| 生产权限 | 乙方部署权限应可审计、可回收；生产密钥由甲方统一管理。 |
| 备份回滚 | 上线前完成数据库备份、容器镜像回滚、固件 OTA 回滚和 DNS 回切方案。 |

8.1 甲方需准备材料汇总

一个可备案主域名及 DNS 管理权限，或明确使用现有域名的子域名授权。

微信小程序账号、微信支付商户号和对应管理员授权。

DeepSeek/大模型、ASR、TTS、声音克隆、内容审核等第三方账号或授权乙方代开。

小程序隐私协议、用户协议、声音克隆授权文案、儿童隐私保护文案和客服入口。
